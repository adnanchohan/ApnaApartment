### **3. Project Scope & Deliverables**

#### **A. User Facing Website (ApnaApartments.com)**
*   **Property Listings & Visual Detail Cards**: 
    *   Apartment Title, Price (formatted in PKR Lakhs/Crores), Rent/Buy tags, Bed/Bath configurations, Area (Sq Ft), exact location string, details description, image carousels, floor number, furnished status, and verified tags.
*   **Dynamic Search Filters**: 
    *   Combined filtering logic based on Location, Rent/Buy, Bedrooms count, Price Range limit, and Furnished status.
*   **Client Contact Options**:
    *   Direct WhatsApp click-to-chat links pre-populated with property titles.
    *   Direct dialer action phone links.
    *   Custom styled contact inquiry forms triggering email notifications.
*   **Interactive Mapping (Optional/Hidden on request)**:
    *   Pre-integrated Leaflet map pins representing property geolocation coordinates, ready to be enabled instantly.
*   **Saved Shortlists**:
    *   A shortlist drawer preserving saved user properties across reloads via browser local storage.

#### **B. Gated Admin Panel & Dashboard**
*   **Property Management Console**:
    *   Forms to Add, Edit, Delete, and Archive apartment listings.
    *   File uploads for managing apartment photos.
*   **Central Settings Console**:
    *   Update primary WhatsApp phone numbers, email notifications routing, and default dialer numbers (Central POC management).
*   **Status Controls**:
    *   Instantly Enable/Disable apartments or entire residential schemes (e.g. DHA, Clifton, Askari) from displaying on the public grid.

#### **C. Hosting & Backend Infrastructure (1-Year Package)**
*   **Cloud Hosting**: Deploying the Node.js/Python server API, frontend build, and admin console on secure, high-uptime cloud infrastructure.
*   **Database Subscription**: Secure storage of apartment details, specs, geolocations, and contact logs.
*   **Media Assets CDN Storage**: 1 year of cloud storage to host high-resolution property pictures.

### **4. Development Timeline (1.5 Months / 6 Weeks)**

During development, the client will be provided with a secure **staging review link** to check real-time progress and test features as they are deployed.

*   **Phase 1: Database Architecture & API Schema Design** *(Week 1)*
    *   Setup relational database schemas, image cloud buckets, and secure admin authentication protocols.
*   **Phase 2: Core Frontend Transition** *(Weeks 2-3)*
    *   Connect the user-facing responsive website to the newly created backend API. Set up live search endpoints.
*   **Phase 3: Admin Console & POC Panel Development** *(Week 4)*
    *   Build the dashboard to upload properties, toggle visibility (Enable/Disable), and manage contact configurations.
*   **Phase 4: Staging Review & Testing** *(Week 5)*
    *   Provide the client with the staging testing link. Conduct QA testing across desktop, tablet, and mobile screens.
*   **Phase 5: Final Production Launch & Handover** *(Week 6)*
    *   Perform server optimization, map domain pointer to ApnaApartments.com, deliver system admin credentials, and launch.

### **5. Revisions & Tweaks Policy**

To ensure we deliver exactly what you envision while maintaining a predictable timeline, we include **three (3) complete rounds of revisions** during the review phases (Phase 4).

> **A Note on Revisions**:  
> We define a "revision" as minor modifications, refinements, and adjustments to existing features or screens that fall within our agreed-upon project scope. Examples include changing styling coordinates (colors, fonts, alignments), modifying button text, adjusting filter options, or shifting image ordering.  
> 
> A revision does not cover the creation of new features, changing the database architecture, or adding integrations outside the initial scope definition. Any new feature requests made after approval will be estimated separately and billed as a project addendum to preserve the launch date.
